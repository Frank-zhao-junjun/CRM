# V6.0 回款跟踪管理 功能规格

## 1. 功能概述

为订单模块增加完整的回款跟踪功能，解决"签单后不知道客户付款状态"的信息孤岛问题。

## 2. 用户痛点

- 销售签单后不清楚客户回款进度
- 财务和销售的回款状态不同步
- 逾期回款无法及时预警

## 3. 功能清单

### 3.1 回款记录管理
- 订单详情页增加「回款记录」Tab
- 支持新增/编辑/删除回款记录
- 回款字段：回款日期、金额、付款方式（银行转账/现金/支票/其他）、备注
- 自动计算已回款金额和待回款金额

### 3.2 回款状态同步
- 订单状态根据回款情况自动更新：
  - 全部回款完成 → 订单自动标记「已付款」
  - 部分回款 → 订单显示「部分回款」
  - 超期未回款 → 触发逾期预警
- 发票状态与回款联动（已收款发票自动标记）

### 3.3 回款统计卡片
- 在订单列表页增加统计：待回款金额、本月待回款、已逾期金额
- 仪表盘增加回款分析模块：
  - 回款率趋势图
  - 逾期账款统计
  - 回款预测

### 3.4 回款提醒
- 超期未回款订单在仪表盘高亮显示
- 支持设置回款到期提醒

## 4. 数据模型

### PaymentReceipt 表
```
- id: UUID
- orderId: UUID (FK)
- amount: Decimal (回款金额)
- paymentDate: Date (回款日期)
- paymentMethod: Enum (银行转账/现金/支票/其他)
- remark: String (备注)
- createdAt: DateTime
- updatedAt: DateTime
```

### 订单表更新
```
- receivedAmount: Decimal (已回款金额，计算得出)
- pendingAmount: Decimal (待回款金额，计算得出)
- paymentStatus: Enum (未回款/部分回款/已回款/逾期)
- dueDate: Date (预计回款日期)
```

## 5. 技术实现

### 5.1 API设计
- `POST /api/orders/{id}/payments` - 新增回款记录
- `GET /api/orders/{id}/payments` - 获取回款记录列表
- `PUT /api/payments/{id}` - 编辑回款记录
- `DELETE /api/payments/{id}` - 删除回款记录
- `GET /api/orders/payments/summary` - 回款统计汇总

### 5.2 组件复用
- 复用订单列表页面结构
- 复用金额显示组件
- 复用状态标签组件
- 复用表单弹窗组件
